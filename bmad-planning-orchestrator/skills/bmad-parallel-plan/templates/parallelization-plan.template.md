This plan turns the ready-for-dev backlog into conflict-free concurrent **waves**. Each
story gets an isolated git worktree branch with a disjoint file scope. Waves run in order;
inside a wave, stories run concurrently. **This is a plan — your external dev tools create
the worktrees, run the agents, merge, review, and open the PR. This plugin does none of that.**

**Merge sequence (ascending story id):**

1. Merge `{{branch_lowest_id}}` -> `integration/wave-{{N}}`
2. Merge next id -> `integration/wave-{{N}}`
3. ... (continue in ascending id order)
4. **Integration review checkpoint** on `integration/wave-{{N}}` — verify the combined
   result (the only place a hidden same-wave semantic interaction would surface).
5. Open PR: `integration/wave-{{N}}` -> `main`.
6. Wave {{N_plus_1}} branches off the merged `main`.

---

## Merge order rationale

- Branches in a wave are file-disjoint, so they merge without textual conflict; the
  ascending-id order is for **predictable, reproducible review**, growing the integration
  branch from the most foundational change outward.
- One reviewed PR per wave (`integration/wave-N -> main`) keeps `main` green instead of
  racing N independent PRs.
- Each wave starts from a known-good `main`, so later waves automatically inherit earlier
  waves' changes and their prerequisites hold.
