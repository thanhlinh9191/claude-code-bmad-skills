## What this plugin is (and is not)

- **Is:** a Claude Code plugin that facilitates BMAD-style **planning** (brief,
  SPEC, PRD, UX, architecture) and **orchestration** (epics, ready-for-dev
  stories, dependency-ordered parallel workstreams, handoff manifests).
- **Is not:** an official BMAD Code Organization product, and not a code-writing
  tool. Implementation is handed off to your own dev tools/plugins. No
  endorsement by the BMAD Code Organization is implied.

## Upstream tracking

This plugin tracks the BMAD Method **v6.x** line. The mapping from each plugin
skill to its upstream BMAD counterpart is documented in
[`resources/bmad-method-mapping.md`](resources/bmad-method-mapping.md), and the
upstream version targeted by each plugin release is recorded in
[`CHANGELOG.md`](CHANGELOG.md).

## Concepts adopted from the BMAD Method

The following BMAD shapes and ideas are used by this plugin and are credited to
the BMAD Code Organization: the four-phase lifecycle (Analysis → Planning →
Solutioning → Implementation); scale-adaptive tracks (Quick Flow / BMad Method /
Enterprise); the `SPEC.md` five-field kernel; three-intent (Create / Update /
Validate) planning skills; the two-document UX contract (`DESIGN.md` +
`EXPERIENCE.md`); the persistent decision log; story-as-compiled-context-object
with source-cited Dev Notes; `project-context.md` as a project "constitution";
the `{epic}.{story}.{slug}.story.md` story format; the implementation-readiness
gate; and `sprint-status.yaml` as the roadmap system-of-record.
